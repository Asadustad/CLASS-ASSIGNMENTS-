ABSTRACTION

  • Abstraction ka matlab hai sirf essential (zaroori) features ko show karna aur unnecessary details ko chhupa dena. Iska maksad code ko simple aur user-friendly banana hota hai.
  • Jese ek car drive karte waqt hume sirf steering, gear aur brake ka use karna hota hai — hume engine ke internal mechanism ka pata nahi hota.

INHERITANCE

  • Inheritance ka matlab hai ek class (child/subclass) doosri class (parent/superclass) ke properties aur methods ko inherit kar sakti hai. Isse code reuse hota hai.
  • Agar ek "Animal" class hai aur "Dog" class us se inherit karti hai, to Dog class mein Animal ke properties/methods automatically aa jate hain.

POLYMORPHISM

  • Polymorphism ka matlab hai "one name, many forms" — yaani ek function ya method different ways mein kaam kar sakta hai. Ye do tareeqon se hota hai:
  • Compile-time (Method Overloading)
  • Run-time (Method Overriding)
  • Ek method speak() jo "Dog" ke liye Bark kare aur "Cat" ke liye Meow.

ENCAPSULATION

  • Encapsulation ka matlab hai ke data aur functions ko ek unit mein band kar dena — yaani class ke andar. Ye kisi bhi object ke internal details ko hide karta hai aur sirf required information ko access karne ki ijaazat deta hai.
  • Class ke andar private variables rakhna aur unko access karne ke liye getter/setter methods banana.
